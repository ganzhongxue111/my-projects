# 技能学习路线图

> 与简历「学习中」技能对应。每完成一项可在简历中将熟练度上调。

## 重要程度说明

| 等级 | 含义 | 学习建议 |
|------|------|----------|
| **P0 核心** | 求职必备，多数目标岗位 JD 直接要求 | 必须掌握，优先做 side project |
| **P1 重要** | 高频使用或面试常问，与 P0 强关联 | 第二阶段深入，能讲清楚原理与场景 |
| **P2 选配** | 加分项、特定岗位才需要、或已有替代方案 | 时间充裕再学，或用到再查 |

**综合优先级（跨阶段 Top 10）：**

1. **P0** LangChain / LangGraph — Agent 编排主战场  
2. **P0** RAG（文档切片 + Embedding + 向量检索）— AI 应用最常见落地形态  
3. **P0** Prompt Engineering + Function Calling — 所有 Agent 的基础  
4. **P0** 大模型 API 调用 — 工程接入入口  
5. **P0** Pandas + SQL — 数据处理岗底线 + Agent 查库场景  
6. **P1** LoRA / LLaMA-Factory — 调优岗核心，应用岗了解即可  
7. **P1** 向量库 Chroma — RAG 标配组件  
8. **P1** Hugging Face 生态 — 微调与模型加载基础  
9. **P1** Cursor Rules / Skills — Vibe Coding 差异化能力  
10. **P2** MCP / Dify / Polars / Ollama — 按目标岗位选学  

---

## 阶段一：数据处理强化（2～3 周）

**目标：** 巩固 Python 数据分析能力，能独立完成 ETL + 可视化 + 简单报表。

| 序号 | 技能 | 重要程度 | 学习资源建议 | 实践项目 |
|------|------|----------|--------------|----------|
| 1 | Pandas 进阶 | **P0 核心** | 《Python for Data Analysis》/ 官方文档 | 清洗一份公开数据集（如 Kaggle） |
| 2 | SQL 进阶 | **P0 核心** | LeetCode SQL / 窗口函数 | 为 side project 设计 star schema |
| 3 | 可视化 | **P1 重要** | Matplotlib + Plotly | 做一份可交互的数据仪表盘 |
| 4 | Polars | **P2 选配** | 官方用户指南 | 用 Polars 重写 Pandas 脚本对比性能 |

**验收标准：** 能写 README 说明数据来源、清洗步骤、结论，并附代码仓库。

---

## 阶段二：大模型 API 与 Prompt（2 周）

**目标：** 熟练调用大模型 API，掌握 Prompt 与 Function Calling。

| 序号 | 技能 | 重要程度 | 学习资源建议 | 实践项目 |
|------|------|----------|--------------|----------|
| 1 | OpenAI / 国产 API | **P0 核心** | 各平台官方 Quickstart | CLI 聊天机器人 |
| 2 | Prompt Engineering | **P0 核心** | OpenAI Prompt Guide | 结构化输出（JSON Schema） |
| 3 | Function Calling | **P0 核心** | LangChain Tools 文档 | 天气 / 计算器 / 查库工具 Agent |

**验收标准：** 一个可调工具的 Python Agent Demo，README 含架构图。

---

## 阶段三：RAG 与 Agent 编排（3～4 周）

**目标：** 搭建文档问答 Agent，理解 RAG 全链路与工作流编排。

| 序号 | 技能 | 重要程度 | 学习资源建议 | 实践项目 |
|------|------|----------|--------------|----------|
| 1 | 文档切片与 Embedding | **P0 核心** | LangChain Document Loaders | PDF/Markdown 知识库 |
| 2 | LangChain / LangGraph | **P0 核心** | 官方 Cookbook | 多步 Agent（检索→总结→回答） |
| 3 | 向量库 Chroma | **P1 重要** | Chroma 官方教程 | 本地向量检索 |
| 4 | MCP | **P2 选配** | Cursor MCP 文档 | 为 Agent 接入 1 个 MCP 工具 |
| 5 | Dify | **P2 选配** | Dify 官方文档 | 无代码复现 RAG 流程 |

**验收标准：** 「个人知识库问答 Agent」项目，可写入简历。

> **说明：** 阶段三整体权重最高——Agent 编排 + RAG 是你四个求职方向中重叠最多、也最容易出作品集的部分。

---

## 阶段四：大模型微调（3～4 周）

**目标：** 完成一次 LoRA 微调，理解数据准备 → 训练 → 评估流程。

| 序号 | 技能 | 重要程度 | 学习资源建议 | 实践项目 |
|------|------|----------|--------------|----------|
| 1 | LLaMA-Factory | **P0 核心** | GitHub README | LoRA 微调 7B 以下模型 |
| 2 | 数据集格式 | **P0 核心** | Alpaca / ShareGPT 格式 | 准备 100～500 条领域 QA |
| 3 | Hugging Face 生态 | **P1 重要** | HF Course | 加载模型做推理 |
| 4 | 评估 | **P1 重要** | 人工对比 + 简单指标 | 微调前后效果对比文档 |
| 5 | Ollama 部署 | **P2 选配** | Ollama 文档 | 本地运行微调后模型 |

**验收标准：** 微调实验报告 + 可复现的训练脚本。

> **说明：** 若主攻「AI 应用 / Agent」而非「算法 / 训练」岗，阶段四可压缩为「跑通一次 LoRA 实验 + 能讲原理」，不必深挖 vLLM 等部署细节。

---

## 阶段五：Vibe Coding 工程化（持续）

**目标：** 将 AI 辅助开发变成可复用的工作流。

| 序号 | 技能 | 重要程度 | 学习资源建议 | 实践项目 |
|------|------|----------|--------------|----------|
| 1 | AI 驱动开发（Cursor 日常） | **P0 核心** | 日常用 Cursor 做 side project | 记录效率对比 |
| 2 | Cursor Rules / Skills | **P1 重要** | Cursor 官方文档 | 为本 repo 写 AGENTS.md |
| 3 | 自动化（GitHub Actions） | **P2 选配** | GitHub Actions + Python | 定时跑数据处理 / 评测脚本 |

**验收标准：** 本 `job hunting` 仓库本身即为作品集之一。

## 建议学习顺序（时间紧时）

```
阶段二（API + Prompt + Tools）→ 阶段三（RAG + LangGraph）→ 阶段一（Pandas/SQL 查漏补缺）
        ↓
阶段四（LoRA 一次实验）→ 阶段五（贯穿全程，用 Cursor 做所有项目）
```

**最小可行作品集（约 6～8 周）：** 一个 RAG 文档问答 Agent（阶段二+三）+ 一次 LoRA 微调实验（阶段四核心项）+ 本仓库 README 展示 Vibe Coding 过程。
